What is a Security Token?

A Security Token is a free, open standard that describes how to build transfer restricted securities tokens on the Ethereum blockchain. While most tokens are have no limitations on who can receive, or participate in the token sale, the Security Token restricts token transfers to verified investors only.

How do they differ from utility tokens?

The concept of utility tokens are fairly well understood in the blockchain space today. Utility tokens represent access to a network, and your token purchase represents the ability to buy goods or services from that network — kind of like an arcade token being used to play a game without owning it. Utility tokens give you that same type of access to a product or service.

On the other hand, security tokens are designed to represent complete or fractional ownership interests (like shares or having stock) in entities. For example, having stake in a private company, public company, real estate or intellectual property.

Overall, security tokens give the holder an ownership right (for company’s share, for example), while utility tokens have a specific use case inside the company’s platform (product or service). Recent involvement of regulation is proof that security tokens are making a movement to become an equal partner in financial world.

STO vs ICO?

An ICO is an unregulated method by which funds are raised for a new blockchain and cryptocurrency oriented company. An Initial Coin Offering (ICO) is used by companies to bypass the rigorous and regulated capital-raising process required by bankers or VCs. In an ICO, a percentage of the cryptocurrency is sold to early backers of the project in exchange for Fiat or other cryptocurrencies.

An STO is a way to issue security tokens that are backed are backed by tangible assets, whereas, in an ICO utility tokens represent future access to a company’s product or service.

With that being said, the value of a security token can increase or decrease on the basis of the performance value of the asset. Think of an STO as a modernized form of an IPO. For example, if a security token represents a percentage of ownership of a publicly traded company and if the publicly listed share increases in value, so will the security token. A major difference between ICO’s and STO’s is the fact that STO’s abide by regulatory frameworks and compliances set by SEC, unlike the completely unregulated environment of ICOs.

WITHOUT A STANDARD

Security Tokens will never be adopted in any meaningful way if issuers, investors, KYC providers, wallets, exchanges, regulators and developers cannot work together within the same framework.

WITH A STANDARD

Security Tokens can become the default form for all financial securities. In the same way that the ERC-20 standard helped to create the boom in utility tokens, the Security Token standard will help create the boom in security tokens. All participants in the ecosystem will be able to speak the same language and become confident building on this new and exciting technology.

Security Token Standard started as a EIP draft written by @frozeman, @adamdossa, @pabloruiz55, and more at the Security Token Roundtable

This Security Token Standard has since moved out of beta and into a community formalized v1 spec, supported and endorsed by a large number of projects from across the crypto ecosystem. This EIP wouldn't be possible without time and dedication from a number of community members.
